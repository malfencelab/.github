# Contributing to `.github`

Thanks for your interest in contributing! 🛡️

This repo follows Malfence's org-wide contribution rules. Please read the canonical guide first:

→ **[malfencelab/.github · CONTRIBUTING.md](https://github.com/malfencelab/.github/blob/main/CONTRIBUTING.md)**

## Repo-Specific Notes

- **Tech stack:** Markdown only
- **Engineering docs:** [`./docs/`](./docs/)
- **Open issues:** [github.com/malfencelab/.github/issues](https://github.com/malfencelab/.github/issues)

## Quick Workflow

1. Fork (or branch if you're a maintainer).
2. Branch from `main` using `feat/`, `fix/`, or `chore/` prefixes.
3. Commit using [Conventional Commits](https://www.conventionalcommits.org/).
4. Open a PR against `main`. One approval required before merge.

Welcome aboard.
